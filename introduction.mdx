# React Native Sherpa ONNX

React Native Sherpa ONNX is a powerful SDK that brings offline and streaming speech processing capabilities to your React Native applications using [sherpa-onnx](https://github.com/k2-fsa/sherpa-onnx).

## What is React Native Sherpa ONNX?

React Native Sherpa ONNX provides native TurboModule bindings for sherpa-onnx, enabling:

- **Offline Speech-to-Text (STT)**: Transcribe audio files or audio samples without internet connectivity
- **Online (Streaming) Speech-to-Text**: Real-time recognition from microphone or audio streams with partial results and endpoint detection
- **Text-to-Speech (TTS)**: Generate high-quality speech audio from text (batch and streaming modes)
- **Voice Activity Detection (VAD)**: Detect speech segments in audio (coming in v0.7.0)
- **Speaker Diarization**: Identify and separate different speakers (coming in v0.4.0)
- **Speech Enhancement**: Improve audio quality (coming in v0.5.0)

All features work **completely offline** - no internet connection or cloud services required.

## Key Features

### Production Ready

- **Full TypeScript support** with complete type definitions
- **iOS and Android support** with automated CI/CD
- **Multiple execution providers**: CPU, NNAPI (Android), XNNPACK, Core ML (iOS), and QNN (Android)
- **95% size reduction** compared to v0.2.x with optimized native binaries

### Flexible Model Support

**Speech-to-Text Models:**
- Zipformer/Transducer (icefall)
- Paraformer (fast non-autoregressive)
- Whisper (multilingual)
- NeMo CTC
- SenseVoice (with emotion detection)
- Moonshine, FunASR Nano, Canary, and many more

**Text-to-Speech Models:**
- VITS (Piper, Coqui, MeloTTS)
- Matcha (high-quality acoustic)
- Kokoro (multi-speaker, multi-language)
- KittenTTS (lightweight)
- Pocket (flow-matching)
- Zipvoice (voice cloning)

## Architecture

React Native Sherpa ONNX is built as a TurboModule that:

1. **Native Layer**: C++/Kotlin (Android) and Objective-C++ (iOS) bindings to sherpa-onnx
2. **JavaScript Layer**: TypeScript API with instance-based engine management
3. **ONNX Runtime**: Optimized inference with multiple execution providers
4. **Model Management**: Flexible loading from assets, filesystem, or Play Asset Delivery (Android)

## Example Output

```typescript
// Speech-to-Text
const result = await stt.transcribeFile('/path/to/audio.wav');
console.log(result.text);
// Output: "Hello world, this is a test transcription"

// Text-to-Speech
const audio = await tts.generateSpeech('Welcome to React Native Sherpa ONNX');
console.log(`Generated ${audio.samples.length} samples at ${audio.sampleRate}Hz`);
// Output: Generated 48000 samples at 22050Hz
```
