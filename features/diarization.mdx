## Status

Speaker diarization support is **planned for a future release**. This feature will enable:

- Identification of different speakers in audio
- Speaker segmentation (who spoke when)
- Speaker counting and clustering
- Integration with STT for speaker-attributed transcripts

## Planned API

The diarization module will provide the following functionality:

```typescript
import { initializeDiarization, diarizeAudio } from 'react-native-sherpa-onnx/diarization';

// Initialize diarization with model
await initializeDiarization({
  modelPath: { type: 'asset', path: 'models/diarization-model' },
});

// Perform speaker diarization
const segments = await diarizeAudio('/path/to/audio.wav');
// Returns: [
//   { speakerId: 'speaker_0', start: 0.5, end: 2.3 },
//   { speakerId: 'speaker_1', start: 2.3, end: 5.2 },
//   { speakerId: 'speaker_0', start: 5.2, end: 7.1 },
//   ...
// ]

// Clean up
await unloadDiarization();
```

## Sherpa ONNX Diarization Status

The upstream [sherpa-onnx](https://github.com/k2-fsa/sherpa-onnx) library's diarization support is under active development. Implementation in this React Native wrapper will follow upstream availability.

Once available, the feature will support:

- Speaker embedding models
- Clustering algorithms for speaker identification
- Real-time diarization (streaming mode)
- Configurable number of speakers or automatic detection

## Workarounds

Until diarization is implemented, you can:

1. **Use external services:** Cloud-based diarization APIs (e.g. AWS Transcribe, Google Speech-to-Text)
2. **Pre-process audio:** Use desktop tools to segment speakers before mobile transcription
3. **Manual annotation:** Manually label speaker segments in your application
4. **Single-speaker assumption:** Process audio as single-speaker until diarization is available
