## Status

Source separation support is **planned for a future release**. This feature will enable:

- Voice/speech isolation from mixed audio
- Music removal from recordings
- Background sound separation
- Multi-source audio decomposition
- Vocal extraction from songs

## Planned API

The separation module will provide the following functionality:

```typescript
import { initializeSeparation, separateSources } from 'react-native-sherpa-onnx/separation';

// Initialize separation with model
await initializeSeparation({
  modelPath: { type: 'asset', path: 'models/separation-model' },
});

// Separate audio into voice and background
const sources = await separateSources('/path/to/mixed-audio.wav');
// Returns: [
//   { sourceId: 'voice', outputPath: '/path/to/voice.wav' },
//   { sourceId: 'music', outputPath: '/path/to/music.wav' },
//   { sourceId: 'background', outputPath: '/path/to/background.wav' },
// ]

console.log('Voice extracted to:', sources[0].outputPath);

// Clean up
await unloadSeparation();
```

## Integration with STT

When available, separation will work as a pre-processing step for better transcription:

```typescript
// Future combined workflow
import { separateSources } from 'react-native-sherpa-onnx/separation';

// 1. Separate voice from background music
```

## Separation Types

Future implementation may support different separation modes:

### Voice/Background Separation
```typescript
const sources = await separateSources('/path/to/audio.wav', {
```

### Music Separation
```typescript

  mode: 'music',

// Returns: vocals + drums + bass + other
```

### Custom Separation
```typescript

  targetSources: ['speech', 'music', 'noise'],

```

## Sherpa ONNX Separation Status

The upstream [sherpa-onnx](https://github.com/k2-fsa/sherpa-onnx) library's source separation support is under evaluation. Implementation in this React Native wrapper will follow upstream availability.

Potential separation models include:

- Voice/background separation (speech enhancement)
- Music source separation (vocals, drums, bass, other)
- Noise isolation and removal
- Multi-speaker separation

## Technical Considerations

Source separation models typically:

- Require significant computational resources
- Have larger model sizes than STT/TTS models
- Benefit from GPU acceleration
- May have higher latency than real-time

Mobile implementation will need to balance quality vs performance.

## Workarounds

Until separation is implemented, you can:

1. **Use external tools:** Pre-process audio with desktop separation tools (e.g. Spleeter, Demucs)
2. **Record clean audio:** Encourage recording in quiet environments without background music
3. **Cloud-based separation:** Use cloud APIs for source separation (requires internet)
4. **Manual editing:** Use audio editing apps to manually remove background sounds
5. **Enhancement instead:** Use speech enhancement models (once available) for noise reduction
