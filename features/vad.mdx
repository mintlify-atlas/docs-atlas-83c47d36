<Note>
  Voice Activity Detection (VAD) is **not yet implemented** in React Native Sherpa ONNX. This page serves as a placeholder for future functionality.
</Note>

## Status

VAD support is **coming in a future release**. This feature will enable:

- Detection of speech segments in audio files
- Real-time voice activity detection
- Silence removal and audio segmentation
- Integration with streaming STT

## Planned API

The VAD module will provide the following functionality:

```typescript
import { initializeVAD, detectVoiceActivity } from 'react-native-sherpa-onnx/vad';

// Initialize VAD with model
await initializeVAD({
  modelPath: { type: 'asset', path: 'models/vad-model' },
});

// Detect voice activity in audio file
const segments = await detectVoiceActivity('/path/to/audio.wav');
// Returns: [{ start: 0.5, end: 2.3 }, { start: 3.1, end: 5.2 }, ...]

// Clean up
await unloadVAD();
```

## Sherpa ONNX VAD Support

The upstream [sherpa-onnx](https://github.com/k2-fsa/sherpa-onnx) library **does support VAD**. Once implemented in this React Native wrapper, it will provide:

- Silero VAD models
- Real-time voice activity detection
- Configurable thresholds and window sizes
- Integration with streaming recognition

For now, VAD functionality is available only through the native C++ API.

## Workarounds

Until VAD is implemented, you can:

1. **Pre-process audio:** Use external tools to detect speech segments before transcription
2. **Use streaming STT with endpoint detection:** The [Streaming STT](/features/streaming-stt) API provides endpoint detection which can serve as a basic VAD alternative
3. **Manual segmentation:** Split audio manually or use time-based chunking
