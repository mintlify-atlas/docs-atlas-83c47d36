## Overview

The Model Download Manager provides a comprehensive system for downloading, caching, and managing Sherpa ONNX models. It supports all model categories including TTS, STT, VAD, Diarization, Enhancement, and Separation models.

## Key Features

- Automatic model discovery from GitHub releases
- Resume support for interrupted downloads
- SHA-256 checksum validation
- Progress tracking with speed and ETA
- LRU-based cache management
- Disk space checking
- Retry with exponential backoff

## Functions

### refreshModelsByCategory

Fetch the latest model list for a category from GitHub releases.

```typescript
async function refreshModelsByCategory<T extends ModelMetaBase>(
  category: ModelCategory,
  options?: {
    forceRefresh?: boolean;
    cacheTtlMinutes?: number;
    maxRetries?: number;
    signal?: AbortSignal;
  }
): Promise<T[]>
```

<ParamField path="category" type="ModelCategory" required>
  Model category to refresh.
</ParamField>

<ParamField path="options.forceRefresh" type="boolean" default={false}>
  Force refresh even if cache is fresh.
</ParamField>

<ParamField path="options.cacheTtlMinutes" type="number" default={1440}>
  Cache validity duration in minutes (default: 24 hours).
</ParamField>

<ParamField path="options.maxRetries" type="number" default={3}>
  Maximum retry attempts for failed requests.
</ParamField>

<ParamField path="options.signal" type="AbortSignal">
  AbortSignal for cancellation.
</ParamField>

**Returns**: Promise resolving to array of model metadata.

**Example**

```typescript
import { ModelCategory, refreshModelsByCategory } from 'react-native-sherpa-onnx/download';

// Refresh TTS models
const ttsModels = await refreshModelsByCategory(ModelCategory.Tts, {
  forceRefresh: true
});

console.log(`Found ${ttsModels.length} TTS models`);
```

### downloadModelByCategory

Download a model with progress tracking and validation.

```typescript
async function downloadModelByCategory<T extends ModelMetaBase>(
  category: ModelCategory,
  id: string,
  opts?: {
    onProgress?: (progress: DownloadProgress) => void;
    overwrite?: boolean;
    signal?: AbortSignal;
    maxRetries?: number;
    onChecksumIssue?: (issue: ChecksumIssue) => Promise<boolean>;
  }
): Promise<DownloadResult>
```

<ParamField path="category" type="ModelCategory" required>
  Model category.
</ParamField>

<ParamField path="id" type="string" required>
  Model identifier.
</ParamField>

<ParamField path="opts.onProgress" type="(progress: DownloadProgress) => void">
  Callback for progress updates.
</ParamField>

<ParamField path="opts.overwrite" type="boolean" default={false}>
  Delete existing model before downloading.
</ParamField>

<ParamField path="opts.signal" type="AbortSignal">
  AbortSignal for cancellation.
</ParamField>

<ParamField path="opts.maxRetries" type="number" default={2}>
  Maximum retry attempts.
</ParamField>

<ParamField path="opts.onChecksumIssue" type="(issue: ChecksumIssue) => Promise<boolean>">
  Custom handler for checksum validation failures. Return true to keep file, false to delete.
</ParamField>

**Returns**: Promise resolving to download result with model ID and local path.

**Example**

```typescript
import { ModelCategory, downloadModelByCategory } from 'react-native-sherpa-onnx/download';

const result = await downloadModelByCategory(
  ModelCategory.Tts,
  'sherpa-onnx-vits-piper-en_US-lessac-medium',
  {
    onProgress: (progress) => {
      console.log(
        `${progress.phase}: ${progress.percent.toFixed(1)}% ` +
        `(${(progress.speed! / 1024 / 1024).toFixed(2)} MB/s, ` +
        `ETA: ${progress.eta?.toFixed(0)}s)`
      );
    }
  }
);

console.log('Model downloaded to:', result.localPath);
```

**Features**:
- Automatic resume for interrupted downloads
- SHA-256 checksum validation
- Disk space checking before download
- Progress tracking with speed and ETA
- Retry with exponential backoff
- Archive extraction with progress

### cleanupLeastRecentlyUsed

Automatically remove least recently used models to free disk space.

```typescript
async function cleanupLeastRecentlyUsed(
  category: ModelCategory,
  options?: {
    targetBytes?: number;
    maxModelsToDelete?: number;
    keepCount?: number;
  }
): Promise<string[]>
```

<ParamField path="options.targetBytes" type="number">
  Target amount of bytes to free. Cleanup stops once reached.
</ParamField>

<ParamField path="options.maxModelsToDelete" type="number">
  Maximum number of models to delete.
</ParamField>

<ParamField path="options.keepCount" type="number" default={1}>
  Minimum number of models to keep.
</ParamField>

**Returns**: Array of deleted model IDs.

**Example**

```typescript
import { cleanupLeastRecentlyUsed, ModelCategory } from 'react-native-sherpa-onnx/download';

// Free up ~500MB by deleting old models
const deleted = await cleanupLeastRecentlyUsed(ModelCategory.Tts, {
  targetBytes: 500 * 1024 * 1024,
  keepCount: 2  // Always keep at least 2 models
});

console.log(`Deleted ${deleted.length} models:`, deleted);
```

## Complete Example

```typescript
import {
  ModelCategory,
  refreshModelsByCategory,
  downloadModelByCategory,
  listDownloadedModelsByCategory,
  getLocalModelPathByCategory,
  subscribeDownloadProgress
} from 'react-native-sherpa-onnx/download';

// Subscribe to progress
const unsubscribe = subscribeDownloadProgress((category, modelId, progress) => {
  if (progress.phase === 'downloading') {
    console.log(
      `Downloading ${modelId}: ${progress.percent.toFixed(1)}% ` +
      `(${(progress.speed! / 1024 / 1024).toFixed(2)} MB/s)`
    );
  } else if (progress.phase === 'extracting') {
    console.log(`Extracting ${modelId}: ${progress.percent.toFixed(1)}%`);
  }
});

// Refresh model list
const models = await refreshModelsByCategory(ModelCategory.Tts);
console.log(`Available models: ${models.length}`);

// Download a model
const modelId = 'sherpa-onnx-vits-piper-en_US-lessac-medium';
const result = await downloadModelByCategory(ModelCategory.Tts, modelId);
console.log('Downloaded to:', result.localPath);

// Get local path for use
const path = await getLocalModelPathByCategory(ModelCategory.Tts, modelId);
if (path) {
  // Use model...
}

// List downloaded models
const downloaded = await listDownloadedModelsByCategory(ModelCategory.Tts);
console.log(`Downloaded models: ${downloaded.length}`);

unsubscribe();
```

## Storage Locations

Models are stored in platform-specific document directories:

- **iOS**: `DocumentDirectory/sherpa-onnx/models/{category}/`
- **Android**: `DocumentDirectory/sherpa-onnx/models/{category}/`

Each model is stored in its own directory named by model ID. The directory contains:
- Model files (`.onnx`, config files, tokens, etc.)
- `manifest.json` - Model metadata and usage tracking
- `.ready` - Marker file indicating successful download
