## Overview

Optimizing React Native Sherpa ONNX performance involves balancing model size, accuracy, and inference speed. This guide covers quantization, execution providers, model selection, and best practices.

## Quantization (int8)

Quantization reduces model size and improves inference speed with minimal accuracy loss.

### Benefits

<CardGroup cols={3}>
  <Card title="Size Reduction" icon="compress">
    ~75% smaller models
  </Card>
  <Card title="Faster Inference" icon="bolt">
    2-4x speedup on CPU
  </Card>
  <Card title="Minimal Loss" icon="check">
    Less than 1% accuracy difference
  </Card>
</CardGroup>

### Using Quantized Models

```typescript
import { createSTT } from 'react-native-sherpa-onnx/stt';

const stt = await createSTT({
  modelPath: { type: 'asset', path: 'models/whisper-tiny-en' },
  modelType: 'auto',
  preferInt8: true, // Automatic int8 selection
});
```

### Quantization Options

| `preferInt8` Value | Behavior |
|-------------------|----------|
| `true` | Use int8 model if available, fallback to fp32 |
| `false` | Always use fp32 model |
| `undefined` (default) | Try int8 first, fallback to fp32 |

### File Detection

When both formats exist:
```
models/whisper-tiny/
├── encoder.onnx          # fp32 version
├── encoder.int8.onnx     # int8 version (used when preferInt8=true)
├── decoder.onnx
├── decoder.int8.onnx
└── tokens.txt
```

<Note>
Quantized models provide the best balance for mobile devices. Use fp32 only when maximum accuracy is critical.
</Note>

## Execution Provider Selection

### Android Providers

<Tabs>
  <Tab title="CPU (Default)">
    **Provider**: `cpu`
    
    - ✅ Always available
    - ✅ Consistent behavior
    - ❌ Slower than hardware acceleration
    - **Use when**: Maximum compatibility needed
    
    ```typescript
    const stt = await createSTT({
      modelPath: { type: 'asset', path: 'models/my-model' },
      provider: 'cpu',
    });
    ```
  </Tab>
  
  <Tab title="XNNPACK">
    **Provider**: `xnnpack`
    
    - ✅ CPU-optimized execution
    - ✅ 2-3x faster than plain CPU
    - ✅ No additional setup
    - **Use when**: CPU-only acceleration needed
    
    ```typescript
    import { getXnnpackSupport } from 'react-native-sherpa-onnx';
    
    const support = await getXnnpackSupport();
    
    if (support.canInit) {
      const stt = await createSTT({
        modelPath: { type: 'asset', path: 'models/my-model' },
        provider: 'xnnpack',
      });
    }
    ```
  </Tab>
  
  <Tab title="NNAPI">
    **Provider**: `nnapi`
    
    - ✅ GPU/DSP/NPU acceleration
    - ✅ Available on most Android devices (API 27+)
    - ⚠️ Performance varies by device
    - **Use when**: Hardware acceleration available
    
    ```typescript
    import { getNnapiSupport } from 'react-native-sherpa-onnx';
    
    const support = await getNnapiSupport();
    
    if (support.canInit) {
      const stt = await createSTT({
        modelPath: { type: 'asset', path: 'models/my-model' },
        provider: 'nnapi',
      });
    }
    ```
  </Tab>
  
  <Tab title="QNN (Qualcomm)">
    **Provider**: `qnn`
    
    - ✅ Qualcomm NPU acceleration (Hexagon)
    - ✅ 5-10x speedup on supported devices
    - ❌ Requires manual library setup
    - **Use when**: Maximum performance on Snapdragon devices
    
    ```typescript
    import { getQnnSupport } from 'react-native-sherpa-onnx';
    
    const support = await getQnnSupport();
    
    if (support.canInit) {
      const stt = await createSTT({
        modelPath: { type: 'asset', path: 'models/my-model' },
        provider: 'qnn',
      });
    }
    ```
    
    See [Android Setup - QNN](/guides/android-setup#qnn-qualcomm-npu) for library installation.
  </Tab>
</Tabs>

### iOS Providers

<Tabs>
  <Tab title="CPU (Default)">
    **Provider**: `cpu`
    
    - ✅ Always available
    - ✅ Consistent behavior
    - ❌ Slower than Core ML
    
    ```typescript
    const stt = await createSTT({
      modelPath: { type: 'asset', path: 'models/my-model' },
      provider: 'cpu',
    });
    ```
  </Tab>
  
  <Tab title="Core ML">
    **Provider**: `coreml`
    
    - ✅ Apple Neural Engine (ANE) acceleration
    - ✅ Available on A12+ devices (iPhone XS+)
    - ✅ GPU fallback on older devices
    - **Use when**: Running on iOS devices
    
    ```typescript
    import { getCoreMlSupport } from 'react-native-sherpa-onnx';
    
    const support = await getCoreMlSupport();
    
    if (support.hasAccelerator) {
      // Apple Neural Engine available
      const stt = await createSTT({
        modelPath: { type: 'asset', path: 'models/my-model' },
        provider: 'coreml',
      });
    }
    ```
  </Tab>
</Tabs>

### Provider Selection Strategy

```typescript
import {
  getQnnSupport,
  getNnapiSupport,
  getXnnpackSupport,
  getCoreMlSupport,
} from 'react-native-sherpa-onnx';
import { Platform } from 'react-native';

async function selectBestProvider(): Promise<string> {
  if (Platform.OS === 'ios') {
    const coreml = await getCoreMlSupport();
    return coreml.hasAccelerator ? 'coreml' : 'cpu';
  }
  
  // Android
  const qnn = await getQnnSupport();
  if (qnn.canInit) return 'qnn'; // Best performance
  
  const nnapi = await getNnapiSupport();
  if (nnapi.canInit) return 'nnapi'; // GPU/DSP/NPU
  
  const xnnpack = await getXnnpackSupport();
  if (xnnpack.canInit) return 'xnnpack'; // CPU-optimized
  
  return 'cpu'; // Fallback
}

const provider = await selectBestProvider();
const stt = await createSTT({
  modelPath: { type: 'asset', path: 'models/my-model' },
  provider,
});
```

## Threading Configuration

### Number of Threads

Control CPU thread count for inference:

```typescript
const stt = await createSTT({
  modelPath: { type: 'asset', path: 'models/whisper-tiny' },
  numThreads: 4, // Default: 1
});
```

### Threading Guidelines

| Device Type | Recommended Threads |
|-------------|--------------------|
| Low-end (2-4 cores) | 1-2 |
| Mid-range (4-6 cores) | 2-4 |
| High-end (8+ cores) | 4-6 |

<Warning>
More threads ≠ faster inference. Optimal value depends on model and device. Test different values for your use case.
</Warning>

## Memory Optimization

### Lazy Model Loading

Load models only when needed:

```typescript
let sttEngine: SttEngine | null = null;

async function transcribe(audioPath: string) {
  if (!sttEngine) {
    sttEngine = await createSTT({
      modelPath: { type: 'asset', path: 'models/whisper-tiny' },
      preferInt8: true,
    });
  }
  
  return await sttEngine.transcribeFile(audioPath);
}

// Cleanup when done
async function cleanup() {
  if (sttEngine) {
    await sttEngine.destroy();
    sttEngine = null;
  }
}
```

### Model Unloading

Always destroy engines when finished:

```typescript
try {
  const stt = await createSTT(...);
  const result = await stt.transcribeFile(audioPath);
  return result;
} finally {
  await stt.destroy(); // Free native resources
}
```

### Play Asset Delivery

Use PAD for large models to reduce APK size:

```typescript
import { getAssetPackPath } from 'react-native-sherpa-onnx';

const padPath = await getAssetPackPath('sherpa_models');
if (padPath) {
  // Models installed via PAD - not in base APK
  const stt = await createSTT({
    modelPath: { type: 'file', path: `${padPath}/whisper-medium` },
    preferInt8: true,
  });
}
```

See [Android Setup - PAD](/guides/android-setup#play-asset-delivery-pad) for configuration.

### Compare Providers

```typescript
const providers = ['cpu', 'xnnpack', 'nnapi', 'qnn'];
const results: Record<string, number> = {};

for (const provider of providers) {
  const support = await checkProviderSupport(provider);
  if (!support) continue;
  
  const stt = await createSTT({
    modelPath: { type: 'asset', path: 'models/whisper-tiny' },
    provider,
  });
  
  const start = Date.now();
  await stt.transcribeFile(audioPath);
  results[provider] = Date.now() - start;
  
  await stt.destroy();
}

console.log('Provider benchmarks:', results);
// Example: { cpu: 2500, xnnpack: 1200, nnapi: 800, qnn: 300 }
```

## Best Practices Summary

<Steps>
  <Step title="Choose the Right Model">
    Balance size, accuracy, and speed for your use case. Prefer quantized (int8) models.
  </Step>
  
  <Step title="Select Best Provider">
    Check provider support at runtime. Use QNN > NNAPI > XNNPACK > CPU priority on Android.
  </Step>
  
  <Step title="Configure Threading">
    Start with 2-4 threads. Benchmark to find optimal value for your device.
  </Step>
  
  <Step title="Manage Memory">
    Load models lazily. Always call `destroy()` when finished.
  </Step>
  
  <Step title="Use PAD for Large Models">
    Keep APK size small by using Play Asset Delivery for models >50MB.
  </Step>
  
  <Step title="Benchmark Your Setup">
    Test different configurations on target devices. Real-world performance varies.
  </Step>
</Steps>
